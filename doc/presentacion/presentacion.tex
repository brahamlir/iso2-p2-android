\subsection{Herramientas y comercialización}
\begin{frame}[fragile]
  \frametitle{Herramientas de Android}
  El SDK de Android incluye varias herramientas que facilitan la creación de
  aplicaciones:
  \begin{itemize}
  \item Emulador. \small{ Permite el diseño, depuración y prueba de las aplicaciones en
    el entorno de ejecución de Android}
  \item Plugin ADT.\small{ Es un plugin integrable en Eclipse que facilita la creación de
    aplicaciones Android}
  \item Android Virtual Devices (AVDs).\small{ Son los dispositivos empleados por el
    emulador para ejecutar las aplicaciones}
  \item Dalvik Debug Monitor Service (DDMS).\small{ Está integrada en la máquina virtual
    Dalvik y ayuda en la depuración}
  \item Android Debug Bridge (ADB).\small{ Entre sus funciones más importantes, está
    la de permitir instalar los archivos .apk}
  \item Android Asset Packaging Tool (APPT). \small{Permite crear los archivos .apk}
  \end{itemize}
\end{frame}


\begin{frame}[fragile]
  \frametitle{Comercialización de las aplicaciones}
\small{
Se puede obtener beneficios de su venta o incluyendo publicidad.\\
Necesaria una cuenta de Google Android Market para publicar al
aplicación.\\
Para generar un archivo .apk válido para Android Market se seguirán estos
pasos:
}
\scriptsize{
\begin{enumerate}
\item Situarse en la carpeta principal del proyecto, en la vista de exploración de
proyectos de Eclipse.
\item Pulsar el botón derecho del ratón entrar en Android Tools.
\item Pulsar en Export Signed Aplication Package.
\item Seleccionar el proyecto a Exportar.
\item Crear una nueva Key (seleccionando donde guardarla y una password).
\item Rellenar los datos de la Key para hacer única las aplicaciones. Se necesitará
indicar un alias, una password, una longevidad de clave (máximo 50 años), y
algún dato de la compañía o personal. Esta Key debe ser usada en futuras
versiones de la aplicación, por lo que es importante protegerla.
\item Guardar el archivo final en el destino deseado y con el nombre elegido.
\end{enumerate}
}
\end{frame}

\begin{frame}[fragile]
  \frametitle{Comercialización de las aplicaciones}
Una vez generada la aplicación, se procederá a publicarla en el Android Market.
Se formalizará una cuenta y se pagarán 25\$, (18 \euro aproximadamente):
\begin{enumerate}
\item Ir a \href{http://market.android.com/}{Android Market}
\item Entrar con una cuenta Google (Gmail por ejemplo). Si no se tiene, se crea
una nueva.
\item Introducir los datos acerca de la cuenta (web, teléfono y nick de publicación).
\item Posteriormente, ir a Google CheckOut y rellenar los datos de pago de nuestra
tarjeta de débito o crédito.
\item Cuando la confirmación de la transacción se reciba, se podrá comenzar a
publicar las aplicaciones.
\end{enumerate}
\end{frame}

\section{AndEngine}
\subsection{Introducción}
\begin{frame}[fragile]
  \frametitle{AndEngine}
  \begin{block}{¿Qué es AndEngine?}
    {\LARGE
      \begin{itemize}
      \item \textbf{Framework} utilizado como motor de juegos para Android
      \item Implementación 2D de \textbf{OpenGl}
      \item Distribuido bajo licencia \textbf{LGPL}
      \end{itemize}}
  \end{block}

 Su finalidad, como la de todos los frameworks, es optimizar el
esfuerzo del desarrollador ofreciéndonos una buena base desde donde
comenzar a crear nuestra propia aplicación.
\end{frame}

\begin{frame}[fragile]
  \frametitle{AndEngine}
  \begin{block}{Utilidades}
    \begin{itemize}
    \item \textbf{Engine}. Motor del juego
    \item \textbf{Scene}. Contenedor
    \item \textbf{Textures}. Imágenes en memoria
    \item \textbf{Sprites}. Objetos interactivos
    \item \textbf{Physics}. Motor físico
    \item \textbf{Detección de colisiones}
    \item \textbf{Música y efectos de sonido}
    \end{itemize}
  \end{block}
\end{frame}

\begin{frame}{Conceptos básicos}
  \begin{itemize}
  \item \textbf{BaseGameActivity}. Raiz del juego
  \item \textbf{Engine}. Motor interno del juego
  \item \textbf{IResolutionPolicy}. Parte del EngineOptions
  \item \textbf{Camera}. Define la parte visible de la escena
  \item \textbf{Scene}. Contenedor para los objetos en la escena
  \item \textbf{Entity}. Objeto que puede ser dibujado
  \item \textbf{Texture}. Imagen almacenada en memoria
  \item \textbf{ITextureSource}. Implementacion de ITextureSource-interface
  \item \textbf{TextureRegion}. Usada por Sprites para posicionar imágenes
  \item \textbf{PhysicsConnector}. Motor de físicas integrado en el Engine
  \end{itemize}
\end{frame}

\subsection{Primer contacto: ejemplo simple}

\begin{frame}[fragile]
  \frametitle{Ejemplo AndEngine}
\begin{block}{}
\emph{BaseGameActivity}. De esta clase debe heredar nuestra clase principal:
\end{block}
\scriptsize{
\begin{verbatim}
public class PracIso2 extends BaseGameActivity {
\end{verbatim}
}
Al heredar de \emph{BaseGameActivity} debemos implementar:

\begin{itemize}
\item \emph{onLoadEngine} Encargado de iniciar el engine del juego.
\item \emph{onLoadResources} Reserva memoria para los recursos, imágenes, música,
  fuentes, etc.
\item \emph{onLoadScene} Carga los recursos en la scene.
\item \emph{onLoadComplete} Código a ejecutar una vez terminado los anteriores.
\end{itemize}

\end{frame}
